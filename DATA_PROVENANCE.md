# Data Provenance - Faces of Plants

This document describes how Faces of Plants accesses, processes, and attributes biodiversity data from GBIF and other sources.

## Data Sources

| Source | API Endpoint | License | Usage |
|--------|-------------|---------|-------|
| GBIF | `https://api.gbif.org/v1` | CC0 / CC-BY / CC-BY-NC (varies by dataset) | Occurrence data, species taxonomy |
| iNaturalist | `https://api.inaturalist.org/v1` | CC-BY-NC | Community observations, species photos |
| Encyclopedia of Life | `https://eol.org/api` | CC-BY-NC | Species descriptions, images |

## GBIF API Integration

### Endpoints Used

1. **Occurrence Search** - `GET /v1/occurrence/search`
   - Purpose: Fetch georeferenced plant occurrence records for map display and species distribution analysis
   - Parameters: species name, country, year range, coordinates, media type, basis of record
   - Always filtered to `kingdom=Plantae` and `hasCoordinate=true`
   - Default limit: 200 records per request

2. **Species Lookup** - `GET /v1/species/{key}`
   - Purpose: Retrieve taxonomic information for identified species
   - Returns: Scientific name, common name, classification, threat status

### How Data is Processed

1. **Query Formation**: User search terms are either passed directly or converted via Mistral LLM into structured GBIF API parameters
2. **API Request**: GBIF client sends request with custom User-Agent header
3. **Response Filtering**: Results are filtered for valid coordinates (lat/lng within bounds)
4. **Caching**: Responses are cached in DynamoDB for 1 hour to reduce API load
5. **Display**: Occurrences are rendered on an interactive Leaflet map with clustering

### User-Agent Identification

All GBIF API requests include a custom User-Agent header for proper identification:

```
FaceOfPlants/0.4.1 (https://facesofplants.org; facesofplants@gmail.com)
```

This allows GBIF to contact us if there are any issues with our API usage, rather than blocking access.

## Data Attribution

### In-App Attribution

- **Map Component**: "Data: GBIF.org (CC BY 4.0)" displayed below every map
- **Footer**: "This product uses data from GBIF.org available under a CC BY 4.0 license"
- **About Page**: Detailed explanation of GBIF data usage and citation requirements

### Required Citations

When using data from this platform in publications, cite the original GBIF datasets:

```
GBIF.org (2026) GBIF Occurrence Download. https://doi.org/10.15468/dl.[XXXXX]
```

The specific DOI is generated by GBIF for each download request. For API queries, cite:

```
GBIF.org (2026) GBIF API. https://api.gbif.org
```

## License Compliance

### CC-BY-NC Datasets

Some GBIF datasets are licensed under CC-BY-NC (Non-Commercial). Currently, Faces of Plants:
- Does not monetize user data or access
- Operates as an open-source research/education project
- Does not filter out NC datasets

If commercial use is added in the future, NC-licensed datasets must be excluded via API parameter `license=CC_BY,CC0`.

### Data Publisher Attribution

Each occurrence record retains references to its original data publisher. The full attribution chain is:

```
FaceOfPlants → GBIF → [Original Dataset Publisher]
```

## No Web Scraping

This project uses **only official GBIF REST APIs** (`api.gbif.org/v1`). No web scraping or HTML parsing is performed. All data access is through documented, versioned API endpoints.

## Reproducibility

To reproduce any query made by this application:

1. Use the GBIF API endpoint: `GET https://api.gbif.org/v1/occurrence/search`
2. Pass the same query parameters logged in the application
3. Results will be equivalent (subject to new data additions by GBIF publishers)
